Relaxometry Phantom Generator

Description

This software aims to reconstruct brain relaxometry for T1 and T2 phantom images. All the image processing used in this project is based on structural brain templates.

The currently available version of RelaxoPhantom toolkit is relaxophantom-v1.0.1, which provides:

  • Relaxometry dataset for T2 and T1 images with different noise levels and RF distortions (please see the script phantom-build.sh for more details)
  • Phantoms with 1 mm^3^ of spatial resolution is available
  • Rician noise with 0%, 3% and 9% are available and Radio-frequency coil artifact with 0%, 20% and 40% are available for the phantom reconstructions

To see a simple example of the image data that is created, please download a relaxometry sample. For this particular example, the slice selected is at the middle brain portion in slice 81 (MNI152). All the 180 slices in the MNI152 is reconstructed in Relaxophantom, given a three-dimensional brain relaxometry simulation.

Requirements

As it is, the software needs FSL (5.0) previously installed on your machine. Please, be sure that FSL are installed and exist an environment variable called fsl5.0 in your system.

To check if exist a link for FSL in your machine type

whereis fsl5.0

in a terminal and certificate that FSL 5.0 is already installed in your machine.

If you do not have FSL installed, you can download it (actually, the FSL-5.0 version) directly from official FSL website. To know more about FSL, see more information about it here. The FSL software has a GNU license. Please check it before use it for any purposes. We do not have any responsibility for any problem or bad functioning of the code.

How to use

Simply download this repository (.zip or .tar.gz compress files present on the top o this web page) and run the script phantom-build.sh. The default image parameters are placed in this main script. If you want to change the parameters in order to create a different simulated brain objects, you can change it in the header section in the phantom-build.sh file. Please, choose those parameters wisely.

Note: Make sure that the .sh files have permission to execute. You can change the permission parameters simply typing

chmod a+x <script file>

in your terminal. This terminal command allows the program execution.
